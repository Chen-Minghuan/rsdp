## 后续关注（TODO）

1. **管理后台接口权限控制**
   - `POST /api/v1/admin/vectors/backfill` 当前无权限限制。
   - 项目接入认证授权体系后，应限制为管理员角色才可调用。

2. **同款判定精度提升**
   - 当前重排主要基于元数据完整性与请求参数做简单加分。
   - 后续应引入 AI 视觉标签（风格、主色、材质、场景）的对比匹配，实现更接近“同款判定”的精排逻辑。

3. **ChromaDB 与 PostgreSQL 状态一致性**
   - 目前仅在产品软删除时同步删除 ChromaDB 向量。
   - 若业务后续出现 `inactive`/下架等非删除状态，需要同步更新 ChromaDB metadata，或在检索后二次校验 PostgreSQL 状态。

4. **Embedding API 重试与限流**
   - `EmbeddingService` 调用 DashScope 时无重试机制。
   - 生产环境建议增加指数退避重试与限流保护。
