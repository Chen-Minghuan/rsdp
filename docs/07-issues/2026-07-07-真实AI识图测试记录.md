# 真实 AI 识图测试记录

## 测试目的

验证关闭 AI Mock 后，使用真实 DashScope `qwen3-vl-plus` 模型进行新品图片识别，以及 `multimodal-embedding-v1` 生成图片向量的全流程是否跑通。

## 环境

- 后端：`http://localhost:8081`，profile = `dev`
- AI 模型：`qwen3-vl-plus`（DashScope 兼容 OpenAI 接口）
- Embedding 模型：`multimodal-embedding-v1`
- 配置文件：`server/src/main/resources/application-dev.yml` 中 `rsdp.ai.mock.enabled=false`
- API Key：通过系统环境变量 `DASHSCOPE_API_KEY` 注入；启动命令中先保存原值再 `source deploy/.env`，避免 `.env` 里的占位符覆盖真实 Key

## 测试步骤

1. 修改 `application-dev.yml`，关闭 AI Mock。
2. 重启后端服务，确保 `DASHSCOPE_API_KEY` 为真实 Key。
3. 管理员登录：`admin / admin123`。
4. 调用 `POST /api/v1/products/entry`，上传测试图片 `test_image.png`（100×100 红色方块）。
5. 轮询任务状态 `GET /api/v1/tasks/{taskId}`。
6. 查看后端日志与产品详情。

## 结论

**真实 AI 识图全流程已跑通。**

本次测试图是一张纯色方块，模型对风格/场景/材质均返回 `unknown`，这是符合预期的；换用真实家具产品图后应能返回有效标签。同时 `Embedding` 和 `ChromaDB` 向量写入也验证通过。

## 注意事项

1. `deploy/.env` 中的 `DASHSCOPE_API_KEY` 当前是占位符；使用真实 Key 时务必通过系统环境变量注入，或在 `.env` 中替换为真实值且不要提交到版本控制。
2. 真实模型调用会产生费用并依赖网络，本地无网络或余额不足时会回到 401 / 调用失败状态。
3. 如果后续需要临时切回 Mock（例如断网调试），可将 `application-dev.yml` 中的 `rsdp.ai.mock.enabled` 改回 `true` 并重启。
