## 三、database/ 数据库脚本

| 文件 | 说明 |
|:-----|:-----|
| `V1__init_db.sql` | 初始化 DDL：字典表、rspu_master/rsku_supply、工厂、导入批次、风格库、sys 权限等全部建表 |
| `V1__seed_data.sql` | 字典种子数据：产品类别、14 个风格编码、材质、场景、角色权限等基础字典 |
| `V2__factory_module.sql` | V2 迁移：扩展 excel_import_batch/factory_master，新增工厂映射、交期规则、产能评估、价格列映射等表 |
| `V3__align_sequences.sql` | V3 迁移：8 张 IdType.AUTO 表的 BIGSERIAL 序列对齐到 MAX(id)，防主键冲突，幂等 |
| `V4__project_module.sql` | V4 迁移：收藏夹 user_favorite、设计项目 project、scheme 方案模板化扩展 |
| `V5__order_module.sql` | V5 迁移：订单模块 design_order/design_order_item（价格 AES 加密）+ 系统配置 sys_config |
| `V6__team_module.sql` | V6 迁移：企业团队轻量版，为 sys_user 增加 company_name/group_name 两个文本字段 |
| `seed_style_knowledge.sql` | 风格知识库种子：11 个风格正向案例 + 9 个失败案例 + 搭配公式（脚本生成，仅覆盖客厅） |
| `reset_db.sql` | 数据库重置：DROP 全部表后重建表结构、种子数据并对齐序列，幂等可重复执行 |

> 约定：新增迁移时 `V1__init_db.sql`、`V1__seed_data.sql`、`reset_db.sql` 三处必须同步更新。

## 七、后端（server/）

技术栈：Spring Boot 3.4 + Java 21 + MyBatis-Plus + PostgreSQL + ChromaDB + MinIO + Redis。

### 7.1 server/ 根配置文件

| 文件 | 说明 |

| `Dockerfile` | 后端镜像：eclipse-temurin:21-jre-alpine 运行 jar，暴露 8081 |

### 7.13 mapper/ 数据访问层

全部为 MyBatis-Plus `BaseMapper` 接口，无自定义方法的仅做标准 CRUD，以下逐文件说明（仅标注有扩展方法的 Mapper 的额外能力）：

- `RspuMapper`：rspu_master 标准 CRUD。
- `RspuStyleMapper` / `RspuSceneMapper` / `RspuVariantMapper` / `RspuRelationMapper`：RSPU 风格/场景/变体/关系关联表标准 CRUD。
- `RskuSupplyMapper`：RSKU 批量插入（insertBatchSafe 兜底逐条）及按 RSPU 列表查询可供货 RSKU（selectCapableByRspuIds）。
- `RspuFactoryMappingMapper`：按 RSPU+省份查询生效的工厂映射（selectActiveByRspuAndProvince）。
- `RspuCodeMapper`：按品类+风格分配 RSPU 业务编码流水号。
- `VariantCodeMapper`：按 RSPU 分配变体编码流水号。
- `FactoryMasterMapper` / `FactoryWarehouseMapper` / `FactoryLevelCapabilityMapper`：工厂档案/仓库/能力等级标准 CRUD。
- `FactoryLeadTimeRuleMapper`：查询工厂生效中的交期规则。
- `FactoryProductCapabilityMapper`：按工厂删除/批量插入（冲突忽略）能力档案，并查询能力同步源数据。
- `PriceHistoryMapper`：价格历史标准 CRUD。
- `ProductStyleMatchMapper`：按 RSPU 查询风格匹配结果。
- `StyleCaseMapper` / `StyleElementMapper` / `StyleMatchingFormulaMapper`：风格案例/拆解元素/搭配公式标准 CRUD。
- `RecommendationScoreConfigMapper`：按 configKey 查配置、清除其他默认配置。
- `ProjectMapper` / `SchemeMapper`：项目/方案主表标准 CRUD。
- `SchemeItemMapper`：方案项批量插入（insertBatchSafe 兜底逐条）。
- `SchemeCandidateMapper`：按推荐请求 ID 查询候选清单并支持批量插入。
- `DesignOrderMapper`：设计订单标准 CRUD。
- `DesignOrderItemMapper`：订单明细批量插入（insertBatchSafe 兜底逐条）。
- `ProductCollectionMapper`：产品集列表联查创建人信息。
- `ProductCollectionItemMapper`：按产品集查询/批量插入/整集删除。
- `UserFavoriteMapper`：用户收藏标准 CRUD。
- `DesignerProfileMapper`：按用户 ID 查设计师画像。
- `SysUserMapper` / `SysRoleMapper` / `SysPermissionMapper`：按用户名/角色码/权限码查单条（default 方法封装 QueryWrapper）。
- `SysUserRoleMapper` / `SysUserFactoryMapper`：按用户 ID 查角色 ID 列表/工厂代码列表（数据权限用）。
- `SysRolePermissionMapper` / `SysConfigMapper` / `CategoryDictMapper`：角色权限、系统配置标准 CRUD；字典按 dict_type 查询。
- `ExcelImportBatchMapper`：导入批次标准 CRUD。
- `ExcelImportRowMapper`：按批次 ID 查询行级导入记录。
- `ImageAssetsMapper` / `AiRecognitionMapper` / `AsyncTaskMapper` / `AuditLogMapper`：图片资产、AI 识别记录、异步任务、审计日志标准 CRUD。

### 7.14 dto/ 数据传输对象

按 `request/`（入参）、`response/`（出参）分包，根包存放 AI/视觉识别等跨模块共享结构，`excel/` 存放 EasyExcel 行模型，多数类为 Record 或 Lombok `@Data`。
