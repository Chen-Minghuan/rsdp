## 双层编码体系

### 业务编码格式（展示/报表用）

> 注意：以下编码是**业务展示编码**，用于人工查看、报表、报价单。数据库主键仍使用 UUID，详见「主键与业务编码分离」部分。

#### RSPU（设计原型）

```
[类别2位]-[风格/定位2位]-[流水3位]-[尺寸1位]
```

| 段 | 示例 | 说明 |
|:---|:---|:---|
| 类别 | FS / SF / TB / BS / OF | 座椅、沙发、桌几、吧椅、办公 |
| 风格/定位 | MC / CR / EX / MG | 中古、奶油、总裁级、经理级 |
| 流水号 | 001 ~ 999 | 该风格下第几款 |
| 尺寸档 | S / M / L / X | 小/中/大/特大；吧椅用 6/7/8 表示座高 |

#### RSKU（供应单元）

```
[RSPU ID]-[工厂代码3位]-[材质版本2位]
```

| 段 | 示例 | 说明 |
|:---|:---|:---|
| RSPU ID | FS-MC-001-M | 继承 |
| 工厂代码 | S001 / A004 | S/A/B/C 分级 |
| 材质码 | PE / LE / TN | PE仿藤、皮革、真藤 |

## 铁律

- `rspu_master` 不含工厂、价格、SKU。
- `rsku_supply` 必须外键关联 `rspu_id`。

## 主键与业务编码分离

- **UUID 主键**：`rspu_master.rspu_id`、`rsku_supply.rsku_id` 使用 UUID（如 `RSPU-3EA08390`、`RSKU-16E02830`），保证稳定、唯一、与业务属性解耦。
- **业务编码**：作为可选展示字段（未来可新增 `rspu_code`、`rsku_code`），不替代 UUID 主键。
- **多属性通过关联表表达**：
  - 多风格 → `rspu_style`
  - 多场景 → `rspu_scene`
  - 多尺寸/颜色/材质变体 → `rspu_variant`
  - 多工厂报价 → `rsku_supply`

## 分类语义

- 家装家具：风格驱动。
- 办公家具：职级驱动，材质码 3 位。
- 吧椅：座高 + 功能驱动，RSKU 加第 7 段功能码。
