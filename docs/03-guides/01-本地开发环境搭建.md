## 环境要求

- Windows 10/11 或 Windows Server / macOS / Linux
- Docker Desktop（开启 WSL2 后端，Windows 推荐）
- JDK 21 LTS
- Node.js 20+ + pnpm 9+
- Maven 3.9+
- Git Bash（Windows 推荐，用于执行 `scripts/start-local.sh`）或 CMD（用于 `scripts/start-local.bat`）

## 推荐方式：一键启动脚本

适合实习生/新成员快速搭好环境。

```bash
# 1. 复制并填写环境变量
cp deploy/.env.example deploy/.env
# 编辑 deploy/.env，至少替换以下 <CHANGE_ME>：
#   - DASHSCOPE_API_KEY
#   - RSDP_JWT_SECRET          （Base64 编码的 32 字节密钥）
#   - RSDP_ENCRYPTION_KEY      （Base64 编码的 32 字节密钥）
#   - POSTGRES_PASSWORD / MINIO_ROOT_PASSWORD / REDIS_PASSWORD

# 2. 启动基础设施（PostgreSQL + ChromaDB + Redis + MinIO，不含 Ollama）
make dev

# 3. 一键启动前后端
# Git Bash / WSL / macOS / Linux：
scripts/start-local.sh

# Windows CMD：
scripts/start-local.bat
```

脚本会自动处理以下本地细节：
- 读取 `deploy/.env` 并校验关键密钥是否仍为 `<CHANGE_ME>`
- 强制 `RSDP_JWT_COOKIE_SECURE=false`（本地 http 开发必需）
- 让后端使用 Docker Postgres 的账号密码
- 在新窗口/后台启动后端 `:8081` 和前端 `:5173`

访问：http://localhost:5173

## 只重启后端（不重启前端）

修改后端代码或数据库权限后，通常只需要重启后端：

```bash
# Git Bash / WSL / macOS / Linux：
scripts/restart-backend.sh

# Windows CMD：
scripts/restart-backend.bat
```

这会停止现有后端进程并重新启动 `mvn spring-boot:run`，前端 dev server 保持运行。

## 手动方式

如果你更想分步启动：

```bash
# 1. 复制并填写环境变量
cp deploy/.env.example deploy/.env

# 2. 启动基础设施（默认不含 Ollama）
make dev

# 3. 导出本地开发必需的环境变量
export RSDP_JWT_COOKIE_SECURE=false
export DB_USERNAME=rsdp
export DB_PASSWORD=<你在 .env 中设置的 POSTGRES_PASSWORD>

# 4. 启动后端
cd server && mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 5. 启动前端
cd web && pnpm install && pnpm dev
```

## 切换到本地 Ollama（可选）

MVP 默认使用 DashScope，无需 Ollama。若后续切换到本地模型：

```bash
# 启动包含 Ollama 的基础设施（需要 NVIDIA GPU）
make infra-ai

# 拉取模型
docker exec -it rsdp-ollama ollama pull qwen2.5-vl:7b
docker exec -it rsdp-ollama ollama pull nomic-embed-text
```

> **Windows 无 GPU**：`make infra-ai` 会因 Ollama 的 GPU 段启动失败。无 GPU 时请保持使用 `make dev`（不启动 Ollama），或手动注释 `deploy/docker-compose.yml` 中 Ollama 服务的 `deploy.resources` 段。

## 环境变量说明

| 变量 | 必填 | 说明 |
|:---|:---|:---|
| `DASHSCOPE_API_KEY` | 是 | DashScope API Key，MVP 阶段 AI 识别/以图搜图使用 |
| `RSDP_JWT_SECRET` | 是 | JWT 签名密钥，Base64 编码的 32 字节 |
| `RSDP_ENCRYPTION_KEY` | 是 | AES-256 加密密钥，Base64 编码的 32 字节 |
| `POSTGRES_PASSWORD` | 是 | Docker Postgres 密码 |
| `POSTGRES_USER` | 否 | 默认 `rsdp` |
| `MINIO_ROOT_PASSWORD` | 是 | MinIO 管理员密码（生产/全服务部署使用） |
| `REDIS_PASSWORD` | 是 | Redis 访问密码（生产/全服务部署使用） |
| `RSDP_JWT_COOKIE_SECURE` | 否 | 本地脚本会自动设为 `false`；生产必须为 `true` |

生成强随机密钥：

```bash
openssl rand -base64 32
```

## 常见问题

### `scripts/start-local.sh` 提示密钥未设置

确保 `deploy/.env` 存在，且 `RSDP_JWT_SECRET`、`RSDP_ENCRYPTION_KEY` 已替换为真实值，不是 `<CHANGE_ME>`。

### Docker 启动 Ollama 失败（Windows 无 GPU）

使用 `make dev` 或 `make infra`（默认不启动 Ollama）。MVP 阶段使用 DashScope，不需要本地 Ollama。

### 后端提示数据库连接失败

检查：
1. `make dev` 是否成功启动了 Postgres。
2. `POSTGRES_PASSWORD` 与后端使用的 `DB_PASSWORD` 是否一致。
3. 若使用 `scripts/start-local.sh`，脚本会自动同步；若手动启动，请显式导出 `DB_USERNAME` 和 `DB_PASSWORD`。

### 登录成功但后续接口 401

本地开发必须使用 `RSDP_JWT_COOKIE_SECURE=false`，否则浏览器不会通过 http 发送 Secure Cookie。`start-local.sh` / `start-local.bat` 已自动处理。

### ChromaDB 连接失败

`make dev` 已将 ChromaDB 映射到 `127.0.0.1:8000`，与 `application-dev.yml` 的 `localhost:8000` 一致。若仍失败，请检查容器是否健康启动。
