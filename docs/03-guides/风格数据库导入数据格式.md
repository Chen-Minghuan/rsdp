# 风格数据库导入数据格式规范

> 本规范用于把在 Kimi / ChatGPT 网页端提取好的「风格案例 + 元素 + 搭配公式」导入 RSDP 数据库。  
> 适用对象：运营/设计师/AI Agent 提取后的数据整理人员。  
> 对应数据库表：`style_case`、`style_element`、`style_matching_formula`。

---

## 1. 总体原则

1. **RSDP 不实现提取功能**，只接收**已经人工复核过**的结构化数据。
2. **Kimi 网页端输出格式**应与本规范保持一致，减少二次转换。
3. 所有编码类字段必须来自 `category_dict` 受控词表，否则导入会失败。
4. **推荐用 JSON 导入**：公式包含嵌套数组，Excel 表达较麻烦；如必须用 Excel，按第 5 节模板处理。

---

## 2. 推荐：JSON 导入格式

### 2.1 顶层结构

```json
{
  "version": "1.0",
  "import_batch_id": "BATCH-20260702-001",
  "created_by": "designer-zhang",
  "cases": [
    { ...案例1... },
    { ...案例2... }
  ],
  "formulas": [
    { ...公式1... },
    { ...公式2... }
  ]
}
```

| 字段 | 必填 | 说明 |
|:-----|:----:|:-----|
| `version` | 是 | 固定 `"1.0"`，后续若字段升级可递增。 |
| `import_batch_id` | 是 | 本次导入批次号，方便追溯。 |
| `created_by` | 是 | 导入人/操作人。 |
| `cases` | 是 | 案例数组，每个案例会写入 `style_case` 及其关联的 `style_element`。 |
| `formulas` | 否 | 搭配公式数组，写入 `style_matching_formula`。可以先只导入案例，后续再补公式。 |

---

### 2.2 案例（case）对象

```json
{
  "case_id": "CASE-MC-001",
  "case_name": "中古风客厅三人位组合",
  "style_code": "MC",
  "room_type": "living_room",
  "is_success": true,
  "source_type": "设计杂志",
  "source_url": "https://example.com/mid-century-living",
  "description": "昌迪加尔椅与焦糖棕皮革沙发、亚麻地毯组合，整体呈现温暖中古调性。",
  "image_url": "minio://style-case/CASE-MC-001.jpg",
  "ai_raw_output": "{ ... Kimi 原始输出原样贴这里 ... }",
  "review_status": "已确认",
  "elements": [
    { "type": "category", "value": "休闲椅", "normalized_code": "FS", "is_primary": true, "confidence": "high" },
    { "type": "category", "value": "三人沙发", "normalized_code": "SF", "is_primary": true, "confidence": "high" },
    { "type": "style", "value": "中古风", "normalized_code": "MC", "is_primary": true, "confidence": "high" },
    { "type": "material", "value": "柚木", "normalized_code": "WO", "is_primary": true, "confidence": "high" },
    { "type": "material", "value": "真藤", "normalized_code": "TN", "is_primary": false, "confidence": "mid" },
    { "type": "material", "value": "皮革", "normalized_code": "LE", "is_primary": false, "confidence": "mid" },
    { "type": "color", "value": "焦糖棕", "is_primary": true, "confidence": "high" },
    { "type": "color", "value": "米白", "is_primary": false, "confidence": "mid" },
    { "type": "scene", "value": "客厅", "normalized_code": "living_room", "is_primary": true, "confidence": "high" },
    { "type": "mood", "value": "温暖", "is_primary": true, "confidence": "mid" }
  ],
  "negative_lesson": ""
}
```

#### 字段说明

| 字段 | 必填 | 类型 | 说明 |
|:-----|:----:|:----:|:-----|
| `case_id` | 是 | string | 案例唯一 ID，建议格式 `CASE-{风格码}-{流水3位}`，如 `CASE-MC-001`。 |
| `case_name` | 是 | string | 人类可读名称。 |
| `style_code` | 是 | string | 主风格编码，必须在 `category_dict(dict_type='style')` 中存在。 |
| `room_type` | 否 | string | 空间类型编码，如 `living_room`、`bedroom`、`office_executive`。建议先在 `category_dict` 中维护。 |
| `is_success` | 是 | bool | `true` = 成功案例，`false` = 失败/反面案例。 |
| `source_type` | 否 | string | 来源类型，如「设计杂志」「品牌画册」「学术论文」「内部项目」。 |
| `source_url` | 否 | string | 来源链接或文献 DOI。 |
| `description` | 否 | string | 案例文字说明。 |
| `image_url` | 否 | string | 图片 MinIO 地址或外部 URL。RSDP 导入时只做文本记录，图片需另行上传。 |
| `ai_raw_output` | 否 | string / object | Kimi 原始输出，可原样保留用于审计。 |
| `review_status` | 是 | string | `待复核` / `已确认` / `存疑`。建议导入前统一为 `已确认`。 |
| `elements` | 是 | array | 元素数组，见 2.3 节。 |
| `negative_lesson` | 否 | string | 失败案例必填：说明为什么失败，如「塑料材质破坏中古质感」。成功案例可留空。 |

---

### 2.3 元素（element）对象

```json
{
  "type": "material",
  "value": "柚木",
  "normalized_code": "WO",
  "is_primary": true,
  "confidence": "high",
  "notes": "框架和椅腿均为实木柚木"
}
```

| 字段 | 必填 | 类型 | 说明 |
|:-----|:----:|:----:|:-----|
| `type` | 是 | string | 元素类型：`style`、`category`、`six_dim_A`~`six_dim_F`、`color`、`material`、`scene`、`mood`、`budget_band`、`function`。 |
| `value` | 是 | string | 人类可读值。 |
| `normalized_code` | 否 | string | 受控词表编码。若 `type` 为 `color`、`mood` 等暂无 code 的，可省略；有 code 的（如 `style`、`category`、`material`）建议必填。 |
| `is_primary` | 否 | bool | 是否为主要元素，默认 `false`。 |
| `confidence` | 否 | string | `high` / `mid` / `low`。 |
| `notes` | 否 | string | 补充说明。 |

---

### 2.4 搭配公式（formula）对象

```json
{
  "formula_id": "FORM-MC-LIVING-001",
  "name": "中古风客厅：实木藤编主椅 × 亚麻/皮革辅椅",
  "style_code": "MC",
  "room_type": "living_room",
  "priority": 10,
  "must_have": [
    { "type": "material", "values": ["实木", "藤编"], "role": "primary" }
  ],
  "compatible": [
    { "type": "style", "values": ["MC", "WJ"], "weight": 0.30, "reason": "同属自然复古谱系" },
    { "type": "material", "values": ["亚麻", "皮革"], "weight": 0.25, "reason": "与中古木藤形成质感对比" },
    { "type": "color", "relation": "analogous", "values": ["焦糖棕", "米白", "原木色"], "weight": 0.20, "reason": "大地色系安全区" },
    { "type": "scene", "values": ["客厅", "书房"], "weight": 0.10 }
  ],
  "avoid": [
    { "type": "material", "values": ["塑料", "亚克力"], "reason": "破坏中古质感" },
    { "type": "color", "values": ["荧光色", "高饱和紫"], "reason": "与中古低饱和调性冲突" }
  ],
  "spatial_hint": {
    "seat_height_delta_cm": [25, 30],
    "scale_ratio": [0.8, 1.2],
    "note": "主椅与辅椅座高差控制在 25-30cm"
  },
  "source_case_ids": ["CASE-MC-001", "CASE-MC-005"],
  "negative_case_ids": ["CASE-MC-FAIL-002"],
  "success_count": 12,
  "fail_count": 3,
  "status": "active"
}
```

| 字段 | 必填 | 类型 | 说明 |
|:-----|:----:|:----:|:-----|
| `formula_id` | 是 | string | 公式唯一 ID，建议 `FORM-{风格码}-{空间}-{流水3位}`。 |
| `name` | 是 | string | 公式名称。 |
| `style_code` | 是 | string | 主风格编码。 |
| `room_type` | 否 | string | 适用空间类型。 |
| `priority` | 否 | int | 优先级，数值越大越优先，默认 `0`。 |
| `must_have` | 否 | array | 必须满足的元素条件。 |
| `compatible` | 否 | array | 加分项，每项带 `weight`（0~1）。 |
| `avoid` | 否 | array | 避雷项，命中后大幅降低分数或提示冲突。 |
| `spatial_hint` | 否 | object | 空间/尺寸建议，如座高差、比例。 |
| `source_case_ids` | 否 | array | 支撑本条公式的成功案例 ID 列表。 |
| `negative_case_ids` | 否 | array | 反面教材 ID 列表。 |
| `success_count` / `fail_count` | 否 | int | 统计支持/违反案例数，用于可信度。 |
| `status` | 否 | string | `active` / `draft` / `disabled`，默认 `active`。 |

#### compatible / avoid 子项格式

```json
{
  "type": "material",
  "values": ["亚麻", "皮革"],
  "weight": 0.25,
  "reason": "质感对比"
}
```

| 字段 | 必填 | 说明 |
|:-----|:----:|:-----|
| `type` | 是 | 元素类型，同 `element.type`。 |
| `values` | 是 | 元素值数组，命中其中任意一个即算命中。 |
| `weight` | 是（compatible） | 权重，0~1，所有 `compatible` 权重之和建议等于 1。 |
| `relation` | 否 | 颜色专用：`analogous`（邻近色）、`complementary`（互补色）、`monochrome`（同色系）。 |
| `reason` | 否 | 推荐理由，前端展示用。 |

## 4. 数据写入的数据库映射

| JSON 节点 | 写入表 | 说明 |
|:----------|:-------|:-----|
| `cases[].*` | `style_case` | 顶层字段直接映射。 |
| `cases[].elements[]` | `style_element` | 每条元素关联 `case_id`。 |
| `formulas[].*` | `style_matching_formula` | `formula_json` 字段存储完整公式 JSON。 |

导入接口（待实现）伪代码：

```java
@PostMapping("/import")
public Result<Void> importStyleKnowledge(@RequestBody StyleKnowledgeImportDTO dto) {
    for (CaseDTO c : dto.getCases()) {
        styleCaseMapper.insert(c);
        for (ElementDTO e : c.getElements()) {
            validateCode(e.getType(), e.getNormalizedCode()); // 校验受控词表
            styleElementMapper.insert(e, c.getCaseId());
        }
    }
    for (FormulaDTO f : dto.getFormulas()) {
        validateCode("style", f.getStyleCode());
        styleMatchingFormulaMapper.insert(f);
    }
    return Result.ok();
}
```

## 5. 备选：Excel 导入格式

如果运营同学习惯用 Excel，建议拆成 3 个工作表：

### Sheet 1：cases

| case_id | case_name | style_code | room_type | is_success | source_type | source_url | description | image_url | review_status | negative_lesson |
|:--------|:----------|:-----------|:----------|:-----------|:------------|:-----------|:------------|:----------|:--------------|:----------------|
| CASE-MC-001 | 中古风客厅三人位组合 | MC | living_room | TRUE | 设计杂志 | | ... | | 已确认 | |
| CASE-MC-FAIL-002 | 中古风客厅塑料椅冲突案例 | MC | living_room | FALSE | 内部项目 | | ... | | 已确认 | 塑料材质与实木中古家具冲突 |

### Sheet 2：elements

| case_id | element_type | element_value | normalized_code | is_primary | confidence | notes |
|:--------|:-------------|:--------------|:----------------|:-----------|:-----------|:------|
| CASE-MC-001 | style | 中古风 | MC | TRUE | high | |
| CASE-MC-001 | material | 柚木 | WO | TRUE | high | |

### Sheet 3：formulas

由于公式包含嵌套数组，Excel 中先用单列 JSON 字符串表达：

| formula_id | name | style_code | room_type | priority | formula_json | source_case_ids | negative_case_ids | success_count | fail_count | status |
|:-----------|:-----|:-----------|:----------|:---------|:-------------|:----------------|:------------------|:--------------|:-----------|:-------|
| FORM-MC-LIVING-001 | 中古风客厅... | MC | living_room | 10 | `{ "must_have": [...], ... }` | CASE-MC-001 | CASE-MC-FAIL-002 | 12 | 3 | active |

> Excel 导入时，后端需要把 `formula_json` 字符串解析为 JSONB。

## 6. 导入前自检清单

- [ ] 所有 `style_code` 已在 `category_dict(dict_type='style')` 中定义。
- [ ] 所有 `category`/`material`/`scene` 的 `normalized_code` 已在对应字典类型中定义。
- [ ] `case_id`、`formula_id` 无重复。
- [ ] `is_success=false` 的失败案例必须填写 `negative_lesson`。
- [ ] 每条 `compatible` 的 `weight` 在 0~1 之间。
- [ ] 同一公式下所有 `compatible.weight` 之和建议等于 1（便于直接计算百分比得分）。
- [ ] `ai_raw_output` 若为空，可写 `{}` 或空字符串。

## 7. 输出文件存放位置

Kimi 网页端提取完成后，把文件按批次放到项目本地目录（该目录已被 `.gitignore` 忽略，不会进入 Git）：

```
data/style-knowledge/
├── raw/          # Kimi 原始输出 JSON / Excel（未清洗）
│   └── BATCH-20260702-001.json
├── cleaned/      # 人工复核后的正式导入文件（可选）
│   └── BATCH-20260702-001-cleaned.json
└── images/       # 案例图片原图（导入前暂存，后续可上传 MinIO）
    └── CASE-MC-001.jpg
```

命名建议：

| 文件类型 | 命名示例 |
|:---------|:---------|
| 原始 JSON | `BATCH-{日期}-{序号}.json`，如 `BATCH-20260702-001.json` |
| 清洗后 JSON | `BATCH-{日期}-{序号}-cleaned.json` |
| 案例图片 | `{case_id}.jpg`，如 `CASE-MC-001.jpg` |

> 注意：`data/` 目录已被 `.gitignore` 排除，适合放本地数据；如果要提交示例到仓库，请放到 `database/` 或 `docs/` 下。

## 8. 给 Kimi 网页端的最简输出指令

你可以直接把下面这段发给 Kimi，让它按本格式输出：

```markdown
请把我上传的设计案例整理成以下 JSON 格式。只输出 JSON，不要解释。

要求：
1. case_id 格式为 CASE-{风格2位码}-{流水3位}。
2. style_code 只能取：MC/CR/IT/FR/WJ/NC/BA/IN/MP。
3. 元素中的 normalized_code 必须来自 RSDP 受控词表，不确定时留空。
4. 成功案例 is_success=true，失败案例 is_success=false 并填写 negative_lesson。
5. 搭配公式中的 compatible.weight 总和等于 1。

输出顶层结构：
{
  "version": "1.0",
  "import_batch_id": "手动填写批次号",
  "created_by": "你的名字",
  "cases": [...],
  "formulas": [...]
}

字段规范详见 `docs/03-guides/风格数据库导入数据格式.md`。
```

## 8. 一句话总结

> **Kimi 网页端最终只需要输出一个顶层包含 `cases` 和 `formulas` 的 JSON 文件，元素和公式都按受控词表编码化、人工复核后，通过 RSDP 导入接口写入 `style_case`、`style_element`、`style_matching_formula` 三张表即可。**
