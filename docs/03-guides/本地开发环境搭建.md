## 环境要求

- Windows 10/11 或 Windows Server
- Docker Desktop（开启 WSL2 后端）
- JDK 17+（推荐 21）
- Node.js 20+ + pnpm 9+
- Maven 3.9+
- PostgreSQL 客户端 `psql`（用于初始化数据库，可选）

## 步骤

1. 克隆项目并进入根目录。
2. 复制环境变量示例：
   ```bash
   cp deploy/.env.example deploy/.env
   # 编辑 deploy/.env 设置 DASHSCOPE_API_KEY、MINIO_PASSWORD、ENCRYPT_AES_KEY 等
   ```
3. 启动基础设施容器（含 PostgreSQL）：
   ```bash
   make infra
   ```
4. 等待 PostgreSQL 就绪（约 5-10 秒），然后初始化数据库：
   ```bash
   make init-db
   make seed
   ```
   或手动执行：
   ```bash
   psql -h localhost -U rsdp -d rsdp -f database/init_db.sql
   psql -h localhost -U rsdp -d rsdp -f database/seed_data.sql
   ```
5. 启动后端：
   ```bash
   cd server && mvn spring-boot:run -Dspring-boot.run.profiles=dev
   ```
6. 启动前端：
   ```bash
   cd web && pnpm install && pnpm dev
   ```
7. 访问 `http://localhost:5173`

## 常见问题

### AI 接口返回 401 / 无 API Key

当前 MVP 默认通过 DashScope 调用 `qwen3-vl-plus`，需要在 `deploy/.env` 中配置：

```bash
DASHSCOPE_API_KEY=your-dashscope-api-key
```

或直接在运行后端的 shell 中设置环境变量：

```bash
export DASHSCOPE_API_KEY=your-dashscope-api-key
```

### Ollama 模型未下载

若已切换到本地 Ollama，首次使用需要拉取模型：

```bash
docker exec -it rsdp-ollama ollama pull qwen2.5-vl:7b
docker exec -it rsdp-ollama ollama pull nomic-embed-text
```

### 数据库连接失败

检查 `server/src/main/resources/application-dev.yml` 中的 PostgreSQL 配置：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/rsdp?currentSchema=public&stringtype=unspecified
    username: ${POSTGRES_USER:rsdp}
    password: ${POSTGRES_PASSWORD:rsdp}
```

确保 `POSTGRES_PASSWORD` 与 `deploy/.env` / docker-compose 中一致。同时注意：若操作系统存在同名环境变量（如 Windows 用户级 `POSTGRES_PASSWORD`），Spring Boot 会优先使用环境变量而非配置文件默认值。详见 `docs/07-issues/2026-06-25-PostgreSQL密码认证失败.md`。
