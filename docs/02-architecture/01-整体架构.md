# RSDP 企业级技术架构方案

> 适用于家具产品数字化管理平台，Windows 台式机/服务器本地部署（推荐配备 NVIDIA RTX 独立显卡），支持后期迁移到云端/私有服务器。

### 1.2 各层职责边界

| 服务 | 语言 | 端口 | 核心职责 | 不负责 |
|:-----|:-----|:-----|:---------|:-------|
| **Nginx** | — | 80/443 | 反向代理、静态资源、SSL 终结、限流 | 业务逻辑 |
| **SpringBoot** | Java 21 | 8080 | 全部业务 CRUD、空间计算引擎、三层检索编排、AI 调用编排（Ollama 向量/标签/终审）、图片预处理调度、PPT/Excel 生成、权限控制、审计日志、任务编排 | AI 模型推理 |
| **Ollama** | — | 11434 | 本地托管嵌入模型与 Qwen 2.5-VL 7B，提供 `/api/embeddings` 与 `/api/generate` HTTP 推理接口（Windows + NVIDIA CUDA，无独显时 CPU 回退） | 业务调用编排 |
| **ChromaDB** | — | 8000 | 512 维向量存储、HNSW 索引、余弦相似度检索 | 元数据存储 |
| **PostgreSQL** | 16 | Docker 容器 / 本地服务 | 产品元数据、工厂档案、供应报价、审计日志的持久化 | 向量检索（style_vector 备份） |
| **Redis** | — | 6379 | 异步任务状态缓存、Session、分布式锁、热点数据缓存 | 持久存储 |
| **MinIO** | — | 9000/9001 | 产品图片、导出文件（PPT/Excel）的对象存储，S3 兼容 | — |

### 1.3 数据流向（新品录入为例）

```
员工拖入图片
    │
    ▼
Vue 前端 ──POST /api/v1/products/entry──▶ SpringBoot
                                              │
                    ┌─────────────────────────┤
                    │                         │
                    ▼                         ▼
              MinIO 存图片              PostgreSQL 写入记录
              (返回图片URL)             (status=processing)
                    │                         │
                    │    ┌────────────────────┘
                    │    ▼
                    │   SpringBoot @Async 异步任务
                    │    │
                    │    ├─┬─▶ Ollama /api/embeddings ──▶ float[512]
                    │    │ │
                    │    │ └─▶ Ollama /api/generate ─────▶ {style, tags, hsv}
                    │    │         (两个 HTTP 调用并行，虚拟线程)
                    │    │
                    │    ├──▶ ChromaDB /query ──▶ Top 5 相似款
                    │    │
                    │    ├──▶ Ollama /api/generate ───▶ {verdict, confidence}
                    │    │
                    │    └──▶ 写入最终结果 ──▶ PostgreSQL UPDATE status=done
                    │
                    ▼
              前端轮询 GET /entry/{taskId} ──▶ 拿到标签面板 ──▶ 员工确认
```

### 1.4 关键设计决策

| 决策 | 说明 |
|:-----|:-----|
| **SpringBoot 直连 Ollama** | SpringBoot 通过 RestClient 直接调用本地 Ollama 的 `/api/embeddings` 与 `/api/generate`，不再经过独立的 AI 服务中间层，架构更简单、部署更少 |
| **AI 推理本地化解耦** | 嵌入模型与 Qwen 2.5-VL 均由 Ollama 本地托管；SpringBoot 负责调用编排，Ollama 只提供 HTTP 推理，不连接数据库、不维护业务状态 |
| **ChromaDB 独立容器** | 与 SpringBoot 解耦，通过 REST API 调用。后续可替换为 Milvus/Qdrant 而不改业务代码 |
| **PostgreSQL 优先** | 开发环境直接使用 PostgreSQL Docker 容器；生产环境可切换独立 PostgreSQL 服务器 |
| **异步任务不阻塞 HTTP** | 图片上传后立即返回 taskId，AI 推理（5-15s）在后台执行，前端轮询或 WebSocket 拿结果 |

## 三、SpringBoot ↔ Ollama 通信设计

### 3.1 调用链路（完整）

```
前端（Vue 3）
    │ POST /api/v1/products/entry
    │ Content-Type: multipart/form-data
    │ Body: { image: File, category_code: "FS", ... }
    ▼
SpringBoot :8080
    │
    ├─ 1. 校验图片（格式/大小/分辨率）→ 通过
    │
    ├─ 2. 上传图片到 MinIO → 返回 URL
    │      PUT /products/images/{uuid}.jpg
    │
    ├─ 3. 在 PostgreSQL 写入 RSPU 草稿（status=processing）
    │
    ├─ 4. 提交异步任务到 productEntryExecutor（虚拟线程池）
    │      返回给前端: { taskId: "task-xxx", status: "processing" }
    │
    └─ 【异步线程开始】
         │
         ├─ 5a. POST ollama:11434/api/embeddings ────┐
         │     Body: { model, input: image_base64 }   │ 并行
         │     Response: { embedding: [0.12, ...] }    │ （CompletableFuture
         │                                              │   .supplyAsync）
         ├─ 5b. POST ollama:11434/api/generate ──────┘
         │     Body: { model, prompt, images, stream: false }
         │     Response: { response: JSON字符串 }
         │
         ├─ 6. POST chromadb:8000/api/v1/collections/products/query
         │     Body: { query_embeddings: [[0.12,...]], n_results: 10 }
         │     Response: { ids: [...], distances: [...], metadatas: [...] }
         │
         ├─ 7. 如果 distance[0] < 0.25（有候选）:
         │     POST ollama:11434/api/generate
         │     Body: { model, prompt, images, stream: false }
         │     Response: { verdict: "same"|"variant"|"new", confidence: 0.92, reasoning: "..." }
         │
         ├─ 8. 根据 verdict 决定:
         │     same   → 关联已有 RSPU，不新建
         │     variant → 关联已有 RSPU，标注"变体"
         │     new     → 生成新 RSPU ID，写入 PostgreSQL
         │
         ├─ 9. 将 512 维向量写入 ChromaDB
         │     POST chromadb:8000/api/v1/collections/products/upsert
         │
         └─ 10. 更新 PostgreSQL: status=done, 写入 task_result
         【异步线程结束】
```

### 3.2 Ollama 接口详情

#### POST /api/embeddings

```
请求:  {
         "model": "nomic-embed-text" | "mxbai-embed-large" | 兼容图像的嵌入模型,
         "input": "base64://..." | "图片Base64"   // 视嵌入模型是否支持图像
       }
响应:  {
         "embedding": [0.123, -0.045, ...],   // 模型原始维度，业务侧投影/截取到 512 个 float32
         "model": "nomic-embed-text"
       }
错误:  { "error": "model not found", "detail": "..." }
超时:  10s（本地嵌入模型通常 200-800ms，10s 是安全上限）
```

#### POST /api/generate（标签生成）

```
请求:  {
         "model": "qwen2.5-vl:7b",
         "system": "你是家具产品分析专家...",
         "prompt": "请对这张产品图输出：风格、六维标签、主色 HSV、场景标签、OCR 文字...",
         "images": ["<图片Base64>"],
         "stream": false,
         "format": "json"
       }
响应:  {
         "response": "{ \"style\": {...}, \"six_dim_tags\": {...}, ... }",
         "done": true
       }
超时:  15s（Qwen-VL 推理 5-8s + 网络往返）
```

#### POST /api/generate（同款终审）

```
请求:  {
         "model": "qwen2.5-vl:7b",
         "system": "你是家具同款判定专家...",
         "prompt": "新品图与以下 Top 3 参考图是否为同款/变体/新款？\n参考：[{rspu_id, distance, six_dim_tags}]",
         "images": [
           "<新品图Base64>",
           "<参考图1Base64>",
           "<参考图2Base64>"
         ],
         "stream": false,
         "format": "json"
       }
响应:  {
         "response": "{ \"verdict\": \"same\"|\"variant\"|\"new\", \"confidence\": 0.92, \"matched_rspu_id\": \"FS-MC-001-M\", \"reasoning\": \"...\" }",
         "done": true
       }
超时:  20s（多图比对推理更长）
```

### 3.3 容错与降级

| 异常场景 | 降级策略 |
|:---------|:---------|
| **Ollama 超时** | 重试 1 次；仍失败 → 任务标记 `failed`，前端提示"AI 服务繁忙，请稍后重试" |
| **嵌入模型失败** | 标记图片质量差（模糊/分辨率不足/非产品图），提示"请上传清晰的产品白底图" |
| **Qwen 标签置信度低** | 照样返回，但 `confidence: "low"` + 前端用黄色标注"建议人工确认" |
| **ChromaDB 不可用** | 跳过同款检索，直接作为新 RSPU 录入（不阻塞流程），事后补检索 |
| **Ollama OOM** | Ollama 健康检查失败 → SpringBoot 感知 → 不调度新任务，已排队的任务等待恢复 |

### 3.4 服务间通信配置

```java
// Ollama + ChromaDB HTTP 客户端
@Configuration
public class RestClientConfig {

    @Bean
    public RestClient ollamaRestClient() {
        return RestClient.builder()
            .baseUrl("http://ollama:11434")
            .requestFactory(new JdkClientHttpRequestFactory())  // 虚拟线程
            .defaultHeader("X-Service", "springboot")
            .requestInterceptor((req, body, exec) -> {
                req.getHeaders().set("X-Request-Id", UUID.randomUUID().toString());
                return exec.execute(req, body);
            })
            .defaultStatusHandler(HttpStatusCode::is5xxServerError,
                (req, resp) -> { throw new AiServiceException("Ollama 推理异常"); })
            .build();
    }

    @Bean
    public RestClient chromaRestClient() {
        return RestClient.builder()
            .baseUrl("http://chromadb:8000")
            .build();
    }
}
```

## 八、异步任务流（新品录入）

### 8.1 完整任务编排

```java
@Service
public class ProductEntryTask {

    private final AiService aiService;
    private final ChromaService chromaService;
    private final ProductService productService;
    private final TaskService taskService;
    private final MinioService minioService;
    private final RspuIdGenerator idGenerator;

    @Async("productEntryExecutor")  // 虚拟线程池，无需配置线程数
    public void processEntry(String taskId, byte[] imageBytes, ProductEntryReq req) {
        try {
            taskService.update(taskId, "processing", 10, "图片预处理中...");

            // ── Step 1: 上传到 MinIO ──
            String imageUrl = minioService.upload(imageBytes, "products/images/");

            // ── Step 2: 并行调用 Ollama（/api/embeddings + /api/generate）──
            taskService.update(taskId, "processing", 20, "AI 识别中...");
            CompletableFuture<float[]> vectorFuture =
                CompletableFuture.supplyAsync(() -> aiService.encode(imageBytes));
            CompletableFuture<AiLabels> labelFuture =
                CompletableFuture.supplyAsync(() -> aiService.label(imageBytes));

            float[] vector = vectorFuture.get(15, TimeUnit.SECONDS);
            AiLabels labels = labelFuture.get(20, TimeUnit.SECONDS);
            taskService.update(taskId, "processing", 50, "AI 识别完成");

            // ── Step 3: ChromaDB 同款检索 ──
            taskService.update(taskId, "processing", 60, "正在检索相似款...");
            List<SimilarResult> candidates = chromaService.query(vector, 10);
            taskService.update(taskId, "processing", 70, "检索完成");

            // ── Step 4: LLM 终审 ──
            String verdict = "new";
            String matchedRspuId = null;
            String reasoning = "";

            if (!candidates.isEmpty() && candidates.get(0).distance() < 0.25) {
                taskService.update(taskId, "processing", 80, "AI 终审中...");
                JudgeResult judge = aiService.judge(imageBytes,
                    candidates.subList(0, Math.min(5, candidates.size())));
                verdict = judge.verdict();   // "same" | "variant" | "new"
                matchedRspuId = judge.matchedRspuId();
                reasoning = judge.reasoning();
            }

            // ── Step 5: 写入数据库 ──
            taskService.update(taskId, "processing", 90, "正在保存...");
            if ("new".equals(verdict)) {
                String rspuId = idGenerator.generate(
                    labels.categoryCode(), labels.styleLabel()
                );
                productService.createRspu(rspuId, labels, vector, imageUrl, req);
                // 写入向量到 ChromaDB
                chromaService.upsert(List.of(rspuId), List.of(vector),
                    List.of(Map.of("rspu_id", rspuId, "category", labels.categoryCode())));
                matchedRspuId = rspuId;
            } else {
                // 关联已有 RSPU，作为变体标注
                productService.markAsVariant(matchedRspuId, labels, imageUrl);
            }

            // ── Step 6: 完成 ──
            taskService.complete(taskId, TaskResult.builder()
                .verdict(verdict)
                .matchedRspuId(matchedRspuId)
                .aiLabels(labels)
                .candidates(candidates.subList(0, Math.min(5, candidates.size())))
                .reasoning(reasoning)
                .imageUrl(imageUrl)
                .build());

        } catch (TimeoutException e) {
            taskService.fail(taskId, "AI 服务超时，请稍后重试");
        } catch (AiServiceException e) {
            taskService.fail(taskId, "AI 识别失败: " + e.getMessage());
        } catch (Exception e) {
            log.error("录入任务异常: taskId={}", taskId, e);
            taskService.fail(taskId, "系统异常，请联系管理员");
        }
    }
}
```

### 8.2 异常恢复机制

| 步骤 | 失败场景 | 恢复方式 |
|:-----|:---------|:---------|
| MinIO 上传 | 磁盘满 / 网络不通 | 重试 3 次；仍失败 → 任务失败，图片保留在临时目录 |
| Ollama 向量编码 | 模型未加载 / OOM | Ollama 健康检查提前拦截；运行时失败重试 1 次 |

| ChromaDB 检索 | 服务不可用 | 跳过检索，直接作为新 RSPU，事后补偿检索 |
| PostgreSQL 写入 | 磁盘满 | 任务失败，人类介入 |

### 9.1 元数据预筛（Layer 1：PostgreSQL）

```java
@Service
public class RetrievalService {

    /**
     * Layer 1: 元数据预筛
     * 用 SQL 硬条件 10万 → 10-50 条
     * 耗时: < 10ms
     */
    public List<RspuMaster> prefilter(String categoryCode, String styleLabel,
                                       Map<String, String> sixDimConditions) {
        LambdaQueryWrapper<RspuMaster> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(RspuMaster::getStatus, "active")
               .isNull(RspuMaster::getDeletedAt);

        // 硬条件 1: 品类
        if (categoryCode != null) {
            wrapper.eq(RspuMaster::getCategoryCode, categoryCode);
        }
        // 硬条件 2: 风格
        if (styleLabel != null) {
            wrapper.eq(RspuMaster::getStyleLabel, styleLabel);
        }
        // 硬条件 3: 六维标签（JSON 字段查询）
        if (sixDimConditions != null) {
            sixDimConditions.forEach((dim, value) -> {
                wrapper.apply("json_extract(six_dim_tags, '$." + dim + "') = {0}", value);
            });
        }

        return rspuMapper.selectList(wrapper);  // 通常返回 8-50 条
    }
}
```

### 9.2 向量精排（Layer 2：ChromaDB）

```java
/**
 * Layer 2: 向量精排
 * 在预筛结果 ID 列表里，用 ChromaDB 算余弦距离
 * 耗时: 30-80ms
 */
public List<SimilarResult> vectorRank(float[] queryVector, List<String> candidateIds) {
    // ChromaDB 支持 where 过滤：只在预筛结果里算距离
    var body = Map.of(
        "query_embeddings", List.of(queryVector),
        "n_results", Math.min(10, candidateIds.size()),
        "where", Map.of("rspu_id", Map.of("$in", candidateIds))
    );
    ChromaQueryResponse resp = chromaRestClient.post()
        .uri("/api/v1/collections/products/query")
        .body(body)
        .retrieve()
        .body(ChromaQueryResponse.class);

    return resp.toSimilarResults();  // Top 5-10 候选
}
```

### 9.3 LLM 终审（Layer 3：Ollama）

```java
/**
 * Layer 3: LLM 多图终审
 * SpringBoot 直接调用 Ollama /api/generate，Qwen-VL 同时看新品图和 Top 3 参考图，做语义比对
 * 耗时: 5-10s
 */
public JudgeResult llmJudge(byte[] queryImage, List<SimilarResult> topCandidates) {
    if (topCandidates.isEmpty()) {
        return JudgeResult.newProduct();
    }

    String references = topCandidates.stream()
        .limit(3)
        .map(c -> String.format(
            "RSPU=%s, distance=%.3f, six_dim=%s",
            c.rspuId(), c.distance(), c.sixDimTags()
        ))
        .collect(Collectors.joining("\n"));

    String prompt = String.format(
        "你是家具同款判定专家。请判断新品图与以下参考图是否为同款/变体/新款，输出 JSON：verdict、confidence、matched_rspu_id、reasoning。\n%s",
        references
    );

    var images = Stream.concat(
        Stream.of(Base64.getEncoder().encodeToString(queryImage)),
        topCandidates.stream().limit(3).map(SimilarResult::imageBase64)
    ).toList();

    var requestBody = Map.of(
        "model", "qwen2.5-vl:7b",
        "prompt", prompt,
        "images", images,
        "stream", false,
        "format", "json"
    );

    OllamaGenerateResponse resp = aiRestClient.post()
        .uri("/api/generate")
        .body(requestBody)
        .retrieve()
        .body(OllamaGenerateResponse.class);

    return objectMapper.readValue(resp.getResponse(), JudgeResult.class);
}
```

### 9.4 三层编排主方法

```java
/**
 * 完整三层检索
 * 输入: 新品图片
 * 输出: { verdict, matchedRspuId, topCandidates, reasoning }
 * 总耗时: 6-12 秒
 */
public RetrievalResult search(byte[] imageBytes, float[] queryVector,
                               String categoryCode, String styleLabel,
                               Map<String, String> sixDimConditions) {

    // Layer 1: 元数据预筛（PostgreSQL, <10ms）
    List<RspuMaster> prefiltered = prefilter(categoryCode, styleLabel, sixDimConditions);
    log.info("预筛完成: {} → {} 条", "100,000", prefiltered.size());

    if (prefiltered.isEmpty()) {
        return RetrievalResult.noMatch();
    }

    // Layer 2: 向量精排（ChromaDB, 30-80ms）
    List<String> candidateIds = prefiltered.stream()
        .map(RspuMaster::getRspuId).toList();
    List<SimilarResult> candidates = vectorRank(queryVector, candidateIds);
    log.info("精排完成: Top {}", candidates.size());

    // Layer 3: LLM 终审（Ollama, 5-10s）
    JudgeResult judge = llmJudge(imageBytes,
        candidates.subList(0, Math.min(3, candidates.size())));
    log.info("终审完成: verdict={}, confidence={}", judge.verdict(), judge.confidence());

    return RetrievalResult.builder()
        .verdict(judge.verdict())
        .confidence(judge.confidence())
        .matchedRspuId(judge.matchedRspuId())
        .candidates(candidates)
        .reasoning(judge.reasoning())
        .build();
}
```

```java
@Service
public class ChromaService {
    private final RestClient restClient;
    private static final String COLLECTION = "products";

    // 查询最相似的 Top K
    public List<SimilarResult> query(float[] embedding, int topK) {
        var body = Map.of(
            "query_embeddings", List.of(embedding),
            "n_results", topK
        );
        return restClient.post()
            .uri("/api/v1/collections/{name}/query", COLLECTION)
            .body(body)
            .retrieve()
            .body(ChromaQueryResponse.class)
            .toSimilarResults();
    }

    // 在指定 ID 集合内查询
    public List<SimilarResult> queryInSet(float[] embedding, int topK,
                                           List<String> rspuIds) {
        var body = Map.of(
            "query_embeddings", List.of(embedding),
            "n_results", topK,
            "where", Map.of("rspu_id", Map.of("$in", rspuIds))
        );
        return restClient.post()
            .uri("/api/v1/collections/{name}/query", COLLECTION)
            .body(body)
            .retrieve()
            .body(ChromaQueryResponse.class)
            .toSimilarResults();
    }

    // 批量插入向量
    public void upsert(List<String> ids, List<float[]> embeddings,
                       List<Map<String, Object>> metadatas) {
        var body = Map.of(
            "ids", ids,
            "embeddings", embeddings,
            "metadatas", metadatas
        );
        restClient.post()
            .uri("/api/v1/collections/{name}/upsert", COLLECTION)
            .body(body)
            .retrieve()
            .toBodilessEntity();
    }

    // 删除向量
    public void delete(List<String> ids) {
        var body = Map.of("ids", ids);
        restClient.post()
            .uri("/api/v1/collections/{name}/delete", COLLECTION)
            .body(body)
            .retrieve()
            .toBodilessEntity();
    }

    // 获取集合信息
    public long count() {
        return restClient.get()
            .uri("/api/v1/collections/{name}", COLLECTION)
            .retrieve()
            .body(ChromaCollectionInfo.class)
            .count();
    }

    // 确保 collection 存在（启动时调用）
    @PostConstruct
    public void ensureCollection() {
        try {
            restClient.get().uri("/api/v1/collections/{name}", COLLECTION)
                .retrieve().toBodilessEntity();
        } catch (Exception e) {
            restClient.post()
                .uri("/api/v1/collections")
                .body(Map.of("name", COLLECTION,
                    "metadata", Map.of("hnsw:space", "cosine")))
                .retrieve().toBodilessEntity();
            log.info("ChromaDB collection '{}' created", COLLECTION);
        }
    }
}
```

## 十五、一句话总结

> **SpringBoot（Java 21）做业务主后端**——CRUD、空间计算、三层检索编排、AI 调用编排、PPT/Excel 生成、权限审计；**Ollama 本地托管嵌入模型与 Qwen 2.5-VL 7B**，通过 `/api/embeddings` 与 `/api/generate` 直接为 SpringBoot 提供向量编码、标签生成、LLM 多图终审能力，不再设独立的 AI 服务中间层。Docker Compose 一键编排 Nginx + SpringBoot + Ollama + ChromaDB + Redis + MinIO 六个服务。SpringBoot 用 RestClient/虚拟线程直接调用 Ollama，所有确定性业务逻辑与 AI 不确定性推理的编排统一在 Java 后端完成，部署更简单、链路更短。多模态图片管线由 SpringBoot 调度，去背景/OCR/HSV 提取可用 Java OpenCV 或外部服务，Ollama 负责语义向量与标签终审，实现"一张图→一份完整产品档案"的智能录入体验。
