# RSDP 数据库设计与项目实现结合说明

> 本文把多模态录入阶段的数据库设计与实际代码实现对应起来，说明每张表在 SpringBoot + Vue3 + Ollama 架构中怎么被创建、使用、流转。
> 目标：让数据库设计不只是纸面结构，而能直接指导代码落地。

## 二、每张表的实现细节

### 2.1 rspu_master — 款式主档案

**在代码中的定位**

这是系统的核心主数据表。前端看到的产品卡片、详情页、AI 标签面板，底层都对应这张表的一行。

**什么时候创建？**

1. 员工上传图片点击"开始录入"
2. SpringBoot 先写一条 `status='draft'`、`review_status='待复核'` 的记录
3. AI 识别完成后更新字段
4. 人工复核后 `review_status='已确认'`

**状态流转**

```
上传图片
  ↓
draft（草稿）
  ↓
processing（AI 识别中）
  ↓
待复核
  ├── 已确认 → active
  ├── 存疑   → 需人工补充信息
  └── 驳回   → discontinued
```

**Java 实体类建议**

```java
@Data
```

**JSON 字段处理建议**

不要每个 JSON 字段都手写 `ObjectMapper`，可以封装一个基类或工具类：

```java
@Component
public class JsonUtils {
    private static ObjectMapper objectMapper;
    
    @Autowired
    public void setObjectMapper(ObjectMapper om) {
        objectMapper = om;
    }
    
    public static String toJson(Object obj) {
        try {
            return objectMapper.writeValueAsString(obj);
        } catch (JsonProcessingException e) {
            throw new BusinessException("JSON序列化失败");
        }
    }
    
    public static <T> T parseJson(String json, Class<T> clazz) {
        try {
            return objectMapper.readValue(json, clazz);
        } catch (Exception e) {
            throw new BusinessException("JSON解析失败");
        }
    }
}
```

**Service 层核心方法**

```java
public interface RspuService {
    // 创建草稿
    RspuMaster createDraft(ProductEntryReq req, String imageId);
    
    // AI 识别完成后更新
    void updateByAiResult(String rspuId, AiResult result);

    void confirm(String rspuId, RspuConfirmReq req, String operator);

```

**与 ChromaDB 的协同**

- `rspu_master.style_vector` 只是备份，**主向量检索走 ChromaDB**
- 写入流程：Ollama 生成向量 → 同时写入 ChromaDB 和 `rspu_master.style_vector`
- 查询流程：先用 ChromaDB 做向量相似度检索 → 再用 `rspu_id` 回到 SQLite 查元数据

### 2.4 image_assets — 图片资源表

**在代码中的定位**

多模态录入的入口。所有 AI 识别都基于这张表的图片。

**图片上传流程**

```
1. 前端拖拽上传 → SpringBoot 接收 MultipartFile
2. 校验：格式、大小、分辨率
3. 质量检测：是否白底、是否模糊（Java OpenCV 简单实现）
4. 上传到 MinIO 或本地目录
5. 写入 image_assets 表
6. 触发 async_task 异步 AI 识别任务
```

**图片类型处理逻辑**

```java
public enum ImageType {
    WHITE_BG,       // 白底图，可进入 AI 识别
    FACTORY_PHOTO,  // 工厂实拍图，用于 RSKU
    DETAIL,         // 细节图
    SCENE,          // 场景图
    ORIGINAL        // 原图，未处理
}
```

**主图逻辑**

- 一个 RSPU 只能有一张 `is_primary=1` 的白底图
- 设置新主图时，需要把旧的 `is_primary` 更新为 0

```java
@Transactional
public void setPrimaryImage(String rspuId, String imageId) {
    imageAssetsMapper.clearPrimaryByRspuId(rspuId);
    imageAssetsMapper.setPrimary(imageId);
}
```

**AI 处理的入口**

只有 `image_type='white_bg'` 且 `quality_score` 达标的图片才会触发 AI 识别：

```java
public void triggerAiRecognition(String imageId) {
    ImageAssets image = imageAssetsMapper.selectById(imageId);
    
    if (!"white_bg".equals(image.getImageType())) {
        return; // 非白底图不识别
    }
    
    if (image.getQualityScore() != null && image.getQualityScore() < 0.6) {
        return; // 质量太差不识别
    }
    
    taskService.submitAiTask(imageId);
}
```

### 2.5 ai_recognition — AI 识别记录表

**在代码中的定位**

系统的"黑匣子"，记录每次 AI 调用。这是后续优化 prompt、换模型、排查问题最重要的依据。

**什么时候写入？**

SpringBoot 每次调用 Ollama 后，无论成功失败都要写入一条记录。

**调用 Ollama 并记录的示例**

```java
@Service
public class AiService {
    
    private final RestClient ollamaClient;
    private final AiRecognitionMapper aiRecognitionMapper;
    
    public AiResult recognizeImage(String imageId, String rspuId) {
        ImageAssets image = imageAssetsMapper.selectById(imageId);
        byte[] imageBytes = minioService.download(image.getStoragePath());
        String base64 = Base64.getEncoder().encodeToString(imageBytes);
        
        long start = System.currentTimeMillis();
        String endpoint = "http://localhost:11434/api/generate";
        
        Map<String, Object> request = Map.of(
            "model", "qwen2.5-vl:7b",
            "prompt", buildLabelPrompt(),
            "images", List.of(base64),
            "stream", false
        );
        
        String response;
        String status = "success";
        String errorMessage = null;
        
        try {
            response = ollamaClient.post()
                .uri(endpoint)
                .body(request)
                .retrieve()
                .body(String.class);
        } catch (Exception e) {
            response = null;
            status = "failed";
            errorMessage = e.getMessage();
            throw new AiServiceException("AI 识别失败", e);
        } finally {
            // 记录识别日志
            AiRecognition record = new AiRecognition();
            record.setRecognitionId(UUID.randomUUID().toString());
            record.setImageId(imageId);
            record.setRspuId(rspuId);
            record.setModelName("qwen2.5-vl:7b");
            record.setRecognitionType("label");
            record.setEndpoint(endpoint);
            record.setInputData(JsonUtils.toJson(request));
            record.setOutputData(response);
            record.setStatus(status);
            record.setErrorMessage(errorMessage);
            record.setProcessingTimeMs((int)(System.currentTimeMillis() - start));
            aiRecognitionMapper.insert(record);
        }
        
        return parseAiResponse(response);
    }
}
```

**如何利用这张表优化 AI？**

```sql
-- 查询某段时间内 AI 识别失败的记录
SELECT * FROM ai_recognition 
WHERE status = 'failed' 
  AND created_at > datetime('now', '-7 days');

-- 统计各模型的平均耗时
SELECT model_name, AVG(processing_time_ms) 
FROM ai_recognition 
WHERE status = 'success' 
GROUP BY model_name;

-- 查询置信度低的识别，用于人工复核重点
SELECT * FROM ai_recognition 
WHERE confidence = 'low' 
  AND recognition_type = 'label';
```

### 2.6 async_task — 异步任务表

**在代码中的定位**

SpringBoot `@Async` 异步任务的持久化状态。前端通过轮询这张表显示进度条。

**典型任务类型**

```java
public enum TaskType {
    PRODUCT_ENTRY,   // 新品录入 AI 识别
    BATCH_IMPORT,    // Excel 批量导入
    ANCHOR_ENCODE    // 锚点图批量编码
}
```

**前端轮询逻辑**

```javascript
// Vue3 伪代码
async function pollTask(taskId) {
  const result = await api.getTask(taskId);
  progress.value = result.progress;
  
  if (result.status === 'done') {
    // 显示 AI 标签面板
    showAiResult(result.resultData);
  } else if (result.status === 'failed') {
    // 显示错误
    showError(result.errorMessage);
  } else {
    // 继续轮询
    setTimeout(() => pollTask(taskId), 1000);
  }
}
```

**SpringBoot 异步任务示例**

```java
@Service
public class ProductEntryTaskService {
    
    @Async("taskExecutor")
    public void process(String taskId, String rspuId, String imageId) {
        taskService.updateProgress(taskId, 10, "图片质量检测中...");
        
        // 1. 向量编码
        taskService.updateProgress(taskId, 30, "向量编码中...");
        List<Float> vector = aiService.encodeImage(imageId, rspuId);
        chromaService.upsert(rspuId, vector);
        
        // 2. 标签生成
        taskService.updateProgress(taskId, 60, "AI 标签生成中...");
        AiResult labelResult = aiService.recognizeImage(imageId, rspuId);
        rspuService.updateByAiResult(rspuId, labelResult);
        
        // 3. 同款判定
        taskService.updateProgress(taskId, 80, "同款判定中...");
        List<SimilarCandidate> candidates = chromaService.query(vector, 5);
        if (!candidates.isEmpty()) {
            JudgeResult judge = aiService.judgeSimilarity(imageId, candidates);
            rspuService.applyJudgeResult(rspuId, judge);
        }
        
        taskService.complete(taskId, resultData);
    }
}
```

## 三、核心业务流程的数据库操作

### 3.1 新品录入完整流程

```
1. 前端上传图片
   → SpringBoot 保存到 MinIO
   → INSERT image_assets (ai_processed=0, quality_score=?, image_type='white_bg')

2. 前端提交录入请求
   → INSERT rspu_master (status='draft', review_status='待复核')
   → INSERT async_task (status='pending', task_type='product_entry')
   → 返回 { taskId }

3. SpringBoot @Async 开始处理
   → UPDATE async_task SET status='processing', progress=10
   → SELECT storage_path FROM image_assets WHERE image_id=?
   → 调用 Ollama /api/embeddings 生成向量
   → INSERT ai_recognition (recognition_type='encode', ...)
   → UPSERT ChromaDB collection='products'

4. AI 标签生成
   → UPDATE async_task progress=60
   → 调用 Ollama /api/generate 生成标签
   → INSERT ai_recognition (recognition_type='label', ...)
   → UPDATE rspu_master SET six_dim_tags=?, positioning_label=?, color_primary_hsv=?, ...

5. 同款判定
   → UPDATE async_task progress=80
   → ChromaDB query 找 Top 5 相似
   → 调用 Ollama /api/generate 多图比对
   → INSERT ai_recognition (recognition_type='judge', ...)
   → UPDATE rspu_master SET review_status='待复核'

6. 完成
   → UPDATE async_task SET status='done', progress=100
   → 前端轮询到 done，展示标签面板

7. 人工复核
   → 员工确认/修正标签
   → UPDATE rspu_master SET review_status='已确认', six_dim_tags=修正后
   → INSERT audit_log (action='UPDATE', table_name='rspu_master', ...)
```

---

### 3.2 同款判定流程

```
输入：新品图（已转为 512 维向量）
  ↓
Layer 1: 元数据预筛（SQLite）
  SELECT * FROM rspu_master
  WHERE category_code = ?
    AND positioning_label = ?
    AND json_extract(six_dim_tags, '$.A') = ?
    AND deleted_at IS NULL
  ↓
Layer 2: 向量精排（ChromaDB）
  collection.query(embedding=new_vector, n_results=5)
  ↓
Layer 3: LLM 终审（Ollama）
  POST /api/generate
  Body: { 新品图 + 参考图 + 六维标签 + prompt }
  Response: { verdict: 'same'|'variant'|'new', confidence, reasoning }
  ↓
结果处理：
  same/variant → 复用已有 rspu_id，只生成新 rsku_id
  new          → 保留当前 rspu_id
```

---

### 3.3 工厂报价录入流程

```
1. 员工选择 RSPU
   → SELECT * FROM rspu_master WHERE rspu_id=?

2. 选择工厂
   → SELECT * FROM factory_master WHERE status='active'

3. 填写报价信息
   → 校验：rspu_id + factory_code + material_code 是否已存在
   → SELECT COUNT(*) FROM rsku_supply WHERE rspu_id=? AND factory_code=? AND material_code=?

4. 生成 RSKU ID
   → 规则：{rspu_id}-{factory_code}-{material_code}
   → 例：FS-MC-001-M-A004-PE

5. 写入数据库
   → INSERT rsku_supply
   → factory_price 先 AES 加密再写入

6. 更新 RSPU 价格区间
   → SELECT MIN(factory_price), MAX(factory_price) FROM rsku_supply WHERE rspu_id=?
   → UPDATE rspu_master SET budget_range=?, reference_price_band=?
```

## 四、关键实现建议

### 4.1 数据库与 ChromaDB 的数据一致性

**问题**：RSPU 元数据在 SQLite，向量在 ChromaDB，如何保持一致？

**方案**

```java
@Transactional
public void createRspuWithVector(RspuMaster rspu, List<Float> vector) {
    // 1. 先写 SQLite
    rspuMapper.insert(rspu);
    
    // 2. 再写 ChromaDB
    try {
        chromaService.upsert(rspu.getRspuId(), vector, buildMetadata(rspu));
    } catch (Exception e) {
        // ChromaDB 失败不影响 SQLite，但要做补偿
        log.error("向量写入失败", e);
        // 可以记录到一张补偿任务表，后续重试
    }
}
```

**补偿机制**

可以增加一张 `vector_sync_task` 表：

```sql
CREATE TABLE vector_sync_task (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rspu_id TEXT NOT NULL,
    action TEXT,        -- 'upsert' / 'delete'
    status TEXT,        -- 'pending' / 'done' / 'failed'
```

### 4.2 图片质量检测

在写入 `image_assets` 时，先做简单质量检测：

```java
public ImageQuality checkQuality(byte[] imageBytes) {
    Mat mat = Imgcodecs.imdecode(new MatOfByte(imageBytes), Imgcodecs.IMREAD_COLOR);
    
    // 1. 分辨率检查
    if (mat.width() < 512 || mat.height() < 512) {
        return ImageQuality.low("分辨率不足");
    }
    
    // 2. 模糊度检查（拉普拉斯方差）
    Mat gray = new Mat();
    Imgproc.cvtColor(mat, gray, Imgproc.COLOR_BGR2GRAY);
    Mat laplacian = new Mat();
    Imgproc.Laplacian(gray, laplacian, CvType.CV_64F);
    MatOfDouble mean = new MatOfDouble();
    MatOfDouble stddev = new MatOfDouble();
    Core.meanStdDev(laplacian, mean, stddev);
    double variance = Math.pow(stddev.get(0, 0)[0], 2);
    
    if (variance < 100) {
        return ImageQuality.low("图片模糊");
    }
    
    // 3. 白底检测（简化：检查边缘像素颜色）
    boolean isWhiteBg = detectWhiteBackground(mat);
    
    return ImageQuality.ok(variance, isWhiteBg);
}
```

质量差的图片 `ai_processed` 不触发，前端提示重新拍摄。

## 五、常见问题

### Q1：为什么 `style_vector` 既存 SQLite 又存 ChromaDB？

**答**：ChromaDB 是主存储，用于高效向量检索；SQLite 存备份用于：
- 离线场景下的简单相似度计算
- 数据导出时不用连 ChromaDB
- 后续数据迁移/重建向量库时不用重新跑 AI

### Q2：`ai_recognition` 会不会数据量很大？

**答**：会。每款产品可能产生 3-5 条记录（encode/label/judge）。建议：
- 定期归档旧数据
- 保留最近 6 个月的详细记录
- 超过 6 个月的只保留统计摘要

### Q3：RSPU 和 RSKU 的 ID 生成在哪里做？

**答**：建议在 SpringBoot Service 层做：
- RSPU ID：根据 category + positioning + 流水号 + 尺寸生成
- RSKU ID：RSPU ID + factory_code + material_code 拼接

### Q4：字典表数据怎么初始化？

**答**：
- 作为种子数据写入 `database/seed_data.sql`
- 系统启动时如果字典表为空，自动初始化
- 管理员可在后台维护
